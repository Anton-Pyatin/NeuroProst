## 🖥️ Системные требования

### Аппаратное обеспечение:
- **Микроконтроллер**: STM32F103C8 (Blue Pill)
- **АЦП**: ADS1256 с кварцем 7.68 МГц
- **Подключение**: USB-UART (CH340)
- **ПК**: Windows/Linux/Mac с Python 3.7+

### Программное обеспечение:
- **Arduino IDE** 1.8.x или 2.x
- **Python** 3.7 или выше
- **PyCharm** (рекомендуется) или любая IDE

## 🔧 Установка

### ШАГ 1: Установка Arduino IDE

1. Скачайте Arduino IDE: https://www.arduino.cc/en/software
2. Установите пакет STM32:
   - Откройте: **Файл → Настройки**
   - В поле "Дополнительные ссылки для менеджера плат" добавьте:
     ```
     https://github.com/stm32duino/BoardManagerFiles/raw/main/package_stmicroelectronics_index.json
     ```
   - Откройте: **Инструменты → Плата → Менеджер плат**
   - Найдите и установите: **STM32 MCU based boards by STMicroelectronics**

3. Установите библиотеку ADS1256:
   - Откройте: **Скетч → Подключить библиотеку → Управление библиотеками**
   - Найдите: **ADS1256 by Curious Scientist**
   - Установите версию 1.6 или выше

### ШАГ 2: Настройка платы STM32

1. Подключите STM32 к ПК через USB
2. Установите драйвер CH340 (если нужно):
   - Windows: https://sparks.gogo.co.nz/ch340.html
3. В Arduino IDE:
   - **Инструменты → Плата** → Generic STM32F1 series
   - **Board part number** → BluePill F103C8
   - **Upload method** → STM32duino bootloader (или Serial, если используете USB-UART)
   - **Port** → выберите ваш COM-порт

### ШАГ 3: Загрузка прошивки Arduino

1. Откройте файл `ADS1256_Streaming.ino` в Arduino IDE
2. Проверьте настройки в коде:
   ```cpp
   #define SERIAL_BAUDRATE 230400   // Скорость UART
   #define SAMPLES_PER_PACKET 10     // Сэмплов в пакете
   #define ADC_DRATE DRATE_2000SPS   // Частота АЦП
   ```
3. Проверьте подключение пинов (если отличается, измените):
   ```cpp
   #define PIN_DRDY PB10  // Data Ready
   #define PIN_RST  PB11  // Reset
   #define PIN_CS   PB12  // Chip Select
   // SPI2: SCK=PB13, MISO=PB14, MOSI=PB15
   ```
4. Нажмите **Загрузить** (Ctrl+U)
5. Дождитесь успешной загрузки

### ШАГ 4: Проверка работы Arduino

1. Откройте **Инструменты → Монитор порта**
2. Установите скорость: **921600 baud**
3. Вы должны увидеть:
   ```
   === ADS1256 8-Channel Streaming System ===
   Initializing...
   ADS1256 detected. STATUS: 0x30
   Configuration complete!
   --- Ready for streaming ---
   ```
4. Введите команду `STATUS` и нажмите Enter
5. Должен ответить информацией о состоянии

## 📊 Использование GUI

### Основные функции:

#### 1️⃣ **Управление подключением**
- **Connect** - подключиться к устройству
- **Refresh** - обновить список портов
- **▶ Start / ⏹ Stop** - запуск/остановка потока данных
- **⏸ Pause / ▶ Resume** - приостановка отображения (данные продолжают записываться)

#### 2️⃣ **Запись данных**
- **⏺ Record** - начать запись в CSV файл
- Выберите имя файла и путь сохранения
- **⏹ Stop Rec** - остановить запись
- Формат CSV: `Time (s), Ch0 (V), Ch1 (V), ..., Ch7 (V)`

#### 3️⃣ **Настройки АЦП**
- **PGA Gain**: Коэффициент усиления (1x, 2x, 4x, 8x, 16x, 32x, 64x)
  - Для малых сигналов используйте большее усиление
  - Внимание: не превышайте ±VREF/PGA на входе!
- **Data Rate**: Частота дискретизации (2000, 5000, 10000 SPS)
  - 2000 SPS = 250 Гц на канал
  - Выше частота = меньше фильтрация шума

#### 4️⃣ **Настройки отображения**
- **Time Window**: Окно времени на графиках (0.5 - 60 секунд)
- **Channel 0-7**: Включение/выключение отображения канала
- **Show Cursors**: Показать измерительные курсоры
  - Перетаскивайте желтые линии для измерений
- **Auto Scale**: Автомасштабирование всех графиков

#### 5️⃣ **Статистика каналов**
Под каждым графиком отображается:
- **Min**: Минимальное значение (mV)
- **Max**: Максимальное значение (mV)
- **Mean**: Среднее значение (mV)
- **P-P**: Peak-to-Peak (размах) (mV)
- **RMS**: Среднеквадратичное значение (mV)

#### 6️⃣ **Статус подключения**
В нижней строке:
- Packets: Количество принятых пакетов
- Lost: Потерянные пакеты
- Rate: Скорость передачи данных (kB/s)

## 🔍 Устранение проблем

### Проблема: "Cannot communicate with ADS1256"
**Решение:**
- Проверьте подключение SPI (SCK, MISO, MOSI, CS)
- Проверьте питание ADS1256 (3.3V или 5V)
- Проверьте подтяжку CS к VCC через резистор 10k
- Измерьте напряжения мультиметром

### Проблема: Arduino загружается, но GUI не подключается
**Решение:**
- Закройте Serial Monitor в Arduino IDE
- Проверьте, что выбран правильный COM-порт
- Попробуйте уменьшить BAUDRATE в Arduino до 460800
- Переподключите USB кабель

### Проблема: Графики не обновляются / зависают
**Решение:**
- Уменьшите Time Window до 2-3 секунд
- Проверьте "Packets Lost" - если много, снизьте DRATE
- Отключите лишние каналы
- Попробуйте уменьшить SAMPLES_PER_PACKET до 5

### Проблема: Много потерянных пакетов (>5%)
**Решение:**
- Используйте качественный USB кабель (короткий)
- Снизьте BAUDRATE до 460800 или 115200
- Увеличьте SAMPLES_PER_PACKET до 20 (меньше пакетов = меньше накладных расходов)
- Закройте другие программы, использующие порт

### Проблема: Нестабильные показания / шум
**Решение:**
- Проверьте качество питания (развязывающие конденсаторы 0.1 мкФ + 10 мкФ на AVDD/DVDD)
- Используйте экранированные провода для аналоговых сигналов
- Подключите AINCOM к GND
- Включите входной буфер в коде: `adc.setBuffer(1)`
- Увеличьте PGA для малых сигналов
- Снизьте DRATE для лучшей фильтрации

### Проблема: Ошибка импорта в Python
**Решение:**
```bash
# Переустановите библиотеки
pip uninstall pyqt5 pyqtgraph pyserial numpy
pip install pyqt5 pyqtgraph pyserial numpy --upgrade
```

## 🎯 Рекомендации по настройке

### Для ЭЭГ/ЭКГ (биосигналы):
```cpp
#define ADC_DRATE DRATE_2000SPS  // 250 Гц на канал
adc.setPGA(4);                    // Усиление 16x
adc.setBuffer(1);                 // Буфер ON
```
```python
time_window = 5.0  # 5 секунд на экране
```

### Для быстрых сигналов (вибрация, акустика):
```cpp
#define ADC_DRATE DRATE_10000SPS  // 1250 Гц на канал
adc.setPGA(0);                     // Усиление 1x
```
```python
time_window = 1.0  # 1 секунда
```

### Для медленных сигналов (температура, давление):
```cpp
#define ADC_DRATE DRATE_2000SPS   // 250 Гц
#define SAMPLES_PER_PACKET 50     // Большие пакеты OK
```
```python
time_window = 30.0  # 30 секунд
```

## 📝 Примечания

- **Максимальная частота на канал**: ~3750 Гц (при 30000 SPS / 8 каналов)
- **Рекомендуемая частота**: 250-500 Гц на канал для стабильной работы
- **Формат данных**: 24-bit signed integer, ±VREF
- **Разрешение**: ~0.3 мкВ при VREF=2.5V, PGA=1
- **Потребление STM32**: ~50 мА
- **Потребление ADS1256**: ~5 мА
