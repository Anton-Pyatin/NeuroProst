## 1. Оборудование и Распиновка

### Необходимое железо
*   **Микроконтроллер:** STM32F103C8 (Blue Pill)
*   **АЦП:** ADS1256 (с кварцем 7.68 МГц)
*   **Интерфейс:** **Отдельный USB-UART адаптер** (CH340/CP2102)
*   **ПК:** Windows 10

### Распиновка подключения (STM32 → ADS1256)
Используется SPI2 интерфейс STM32.

| STM32 Pin | ADS1256 Pin | Назначение |
| :--- | :--- | :--- |
| **PB13** | SCLK | Тактовая линия SPI |
| **PB14** | DOUT (MISO) | Данные от АЦП |
| **PB15** | DIN (MOSI) | Данные к АЦП |
| **PB12** | CS | Выбор чипа |
| **PB10** | DRDY | Готовность данных (Прерывание) |
| **PB11** | RESET | Сброс (опционально) |
| **3.3V** | AVDD, DVDD | Питание (Обязательно 3.3В!) |
| **GND** | AGND, DGND | Земля |

> ⚠️ **Важно:** Не подавайте 5V на пины ADS1256, это выведет чип из строя. Используйте стабильное питание 3.3V.

### Аналоговые входы
*   **AIN0-AIN7:** Дифференциальные входы.
*   **AINCOM:** Общий провод (подключить к GND для single-ended режима).

## 2. Прошивка STM32 (Arduino IDE)

*Примечание: Прошивка загружается через Arduino IDE. Библиотека ADS1256 не требуется, код работает автономно.*

### Шаг 2.1: Подготовка Arduino IDE
1.  Установите **Arduino IDE** (версия 1.8.x или 2.x).
2.  Добавьте поддержку STM32:
    *   `Файл` → `Настройки`.
    *   В поле "Дополнительные ссылки для менеджера плат" вставьте:
        ```text
        https://github.com/stm32duino/BoardManagerFiles/raw/main/package_stmicroelectronics_index.json
        ```
    *   `Инструменты` → `Плата` → `Менеджер плат` → Найдите `STM32 MCU based boards` → Установите.
3.  **Не устанавливайте** библиотеку `ADS1256 by Curious Scientist` (в новой прошивке не используется).

### Шаг 2.2: Настройка и Загрузка
1.  Откройте файл **`NeuroProst.ino`**.
2.  **Проверьте настройки в коде** (важно для согласования с Python):
    ```cpp
    // Проверьте значение baudrate в коде
    #define SERIAL_BAUDRATE 230400 
    #define SAMPLES_PER_PACKET 25
    #define ADC_DRATE DRATE_2000SPS  // 250 Гц на канал (8 каналов)
    ```
    *> Примечание: Для UART-адаптеров рекомендуется 230400 или 115200. Если в коде стоит 921600, могут быть потери пакетов.*
3.  Подключите STM32 к ПК через UART-адаптер.
4.  В Arduino IDE выберите:
    *   `Плата`: Generic STM32F1 series → BluePill F103C8.
    *   `Порт`: Ваш COM-порт (адаптера).
    *   `Метод загрузки`: STM32duino bootloader (или Serial).
5.  Нажмите **Загрузить** (Ctrl+U).
6.  Откройте **Монитор порта** (выберите baudrate как в коде) и введите `STATUS`, чтобы убедиться, что АЦП обнаружен.

## 3. Настройка проекта в PyCharm

### Шаг 3.1: Импорт проекта
1.  Запустите **PyCharm**.
2.  Нажмите **Open** (или `File` → `Open`).

## 4. Установка библиотек через Python Packages

В PyCharm удобно управлять зависимостями через графический интерфейс.

### Шаг 4.1: Открытие меню пакетов
1.  В нижнем pane окна PyCharm найдите вкладку **Python Packages** (если нет, включите через `View` → `Tool Windows` → `Python Packages`).
2.  Убедитесь, что сверху выбран правильный интерпретатор проекта.

### Шаг 4.2: Установка зависимостей
1.  В строке поиска введите название библиотеки и нажмите `+` (Install) рядом с нужной версией.
2.  Установите следующие пакеты по очереди:
    *   `pyqt5`
    *   `pyqtgraph`
    *   `pyserial`
    *   `numpy`
    *   `pandas`
    *   `scipy`
    *   `pyinstaller` (для сборки EXE)
3.  Дождитесь окончания установки для каждого пакета (статус изменится на `Installed`).

> **Альтернатива:** Если у вас есть файл `requirements.txt`, PyCharm может предложить установить их автоматически при открытии проекта. Нажмите `Install Requirements`, если появится всплывающее уведомление.

## 5. Запуск GUI через PyCharm

### Шаг 5.1: Конфигурация запуска
1.  В проекте откройте файл `Python/ads1256_gui.py` (или просто `ads1256_gui.py` в корне).
2.  В правом верхнем углу нажмите на выпадающий список конфигураций → `Edit Configurations`.
3.  Нажмите `+` → `Python`.
4.  Заполните поля:
    *   **Name:** `NeuroProst GUI`
    *   **Script path:** Выберите файл `ads1256_gui.py`.
    *   **Working directory:** Укажите папку проекта (где лежит файл `.py`). **Это критично!**
5.  Нажмите `OK`.

### Шаг 5.2: Первый запуск
1.  Убедитесь, что STM32 подключен и прошит.
2.  **Закройте Монитор порта Arduino IDE** (порт будет занят, и Python не сможет подключиться).
3.  В PyCharm нажмите зеленую кнопку **Run** (Shift+F10).
4.  Должно открыться окно приложения.

## 6. Сборка EXE файла

Для передачи программы на другие ПК без установки Python.

### Шаг 6.1: Подготовка
Убедитесь, что в проекте есть файл `build_exe.py`. Его содержимое должно выглядеть примерно так:
```python
import PyInstaller.__main__

PyInstaller.__main__.run([
    'ads1256_gui.py',
    '--onefile',
    '--windowed',
    '--name=NeuroProst_Recorder',
    '--clean'
])
```

### Шаг 6.2: Настройка запуска сборки
1.  Откройте файл `build_exe.py` в PyCharm.
2.  Создайте новую конфигурацию (`Edit Configurations` → `+` → `Python`).
3.  **Name:** `Build EXE`
4.  **Script path:** `build_exe.py`
5.  **Working directory:** Папка проекта.
6.  Сохраните.

### Шаг 6.3: Запуск сборки
1.  Выберите конфигурацию `Build EXE` в верхнем меню.
2.  Нажмите **Run**.
3.  В окне **Run** внизу вы увидите процесс сборки.
4.  После завершения в папке проекта появится папка `dist`, внутри которой будет `NeuroProst_Recorder.exe`.

> ⚠️ **Антивирус:** Собранный EXE файл может быть ложно заблокирован антивирусом Windows 10. Добавьте папку `dist` в исключения.

## 7. Использование системы

### Подключение
1.  Запустите GUI (через PyCharm или готовый EXE).
2.  Выберите **COM-порт** (список обновляется кнопкой 🔄).
3.  Выберите **Baudrate**: `230400` (должно совпадать с прошивкой `NeuroProst.ino`).
4.  Нажмите **Connect**. Статус внизу должен стать зеленым.

### Запись данных
1.  Нажмите **▶ Start** — графики начнут обновляться.
2.  Нажмите **⏺ Record** — выберите путь для сохранения CSV.
3.  Для остановки записи нажмите **⏹ Stop Rec**.
4.  Для остановки потока данных нажмите **⏹ Stop**.

### Настройки сигналов (Пресеты)
| Тип сигнала | PGA (Gain) | Data Rate | Time Window | Filter |
| :--- | :--- | :--- | :--- | :--- |
| **ЭЭГ/ЭКГ** | 16x | 2000 SPS | 5 сек | Lowpass 50 Hz |
| **Вибрация** | 1x | 10000 SPS | 1 сек | Highpass 10 Hz |
| **Температура** | 1x | 2000 SPS | 30 сек | Lowpass 1 Hz |

## 8. Устранение проблем

### Проблема: "Cannot communicate with ADS1256" / Нет связи
*   **Проверка питания:** Убедитесь, что на AVDD/DVDD ровно 3.3V.
*   **Проверка SPI:** Проверьте контакты PB13-PB15.
*   **Проверка DRDY:** Пин PB10 должен генерировать импульсы.

### Проблема: GUI не подключается (Port Busy)
*   **Причина:** Порт занят другой программой.
*   **Решение:** Закройте Arduino IDE (Serial Monitor) и любые другие терминалы. Переподключите USB-адаптер.
*   **Драйверы:** Убедитесь, что драйвер CH340/CP2102 установлен в Windows 10 (Проверьте в Диспетчере устройств).

### Проблема: Много потерянных пакетов (Lost Packets > 5%)
*   **Решение 1:** Используйте качественный короткий USB кабель для UART-адаптера.
*   **Решение 2:** В прошивке `NeuroProst.ino` уменьшите `SERIAL_BAUDRATE` до 115200.
*   **Решение 3:** Увеличьте `SAMPLES_PER_PACKET` (например, до 50), чтобы снизить накладные расходы.
*   **Решение 4:** В GUI уменьшите `Time Window` до 2-3 секунд.

### Проблема: Ошибки импорта в PyCharm
*   **Решение:** Откройте вкладку **Python Packages**. Найдите проблемную библиотеку (например, PyQt5) → нажмите значок корзины (Uninstall) → установите заново через `+`.
*   **Альтернатива:** В терминале PyCharm: `pip install --upgrade pyqt5 pyqtgraph pyserial`.

### Проблема: EXE не запускается на другом ПК
*   **Драйвер:** Установите драйвер CH340 на целевом ПК.
*   **Зависимости:** Убедитесь, что при сборке не было ошибок (смотрите логи в PyCharm).
*   **Windows:** Требуется Windows 7 SP1 или новее. Установите Visual C++ Redistributable.
